# Contributing to bash.d

Thanks for thinking of contributing! Please follow these guidelines to help the project stay healthy and maintainable.

## Quickstart
1. Fork the repo
2. Create a short-lived branch: `git checkout -b feat/my-task`
3. Add tests where applicable
4. Run existing tests and add new tests if needed
5. Create a PR and reference the related task in `tasks.md` or `master_todo.md`

## Code Style
- Use consistent naming and style conventions (see `rules.md`)
- Keep commits small and descriptive
- Write tests for bug fixes and new features

## Reviews & CI
- PRs must pass the test suite and any configured linters
- Add helpful description, screenshots or examples where relevant
- Tag reviewers appropriately
